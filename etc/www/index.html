task

Version 0.1

Task is a simple management cli for tracking tasks.

This interface wraps a file using the simplest possible format where each task is represented as a single line of text in the file.

Each task has an id of its line number in the file.

Task ids are ephemeral, so as tasks are completed, added or prioritised the id will change. The id is primarily for interactive use.

quick start

Here are the basic operation you can perform with task. These examples use long form commands, however you can replace any long form command with its first letter, i.e. l is equivalent to list, n is equivalent to new:

specify a custom task file: 

    $ TASK_FILE=~/.task task

list all tasks: 

    $ task

search in list of tasks: 

    $ task list searchterm

complete task at top of stack:

    $ task mark

complete a specific task:

    $ task mark 2

create task:

    $ task new This is an item in my task list.

order tasks in editor:

    $ task order

order tasks in specific editor:

    $ EDITOR=emacs task order

move a task to the top of the stack:

    task prioritise task-id

list all complete tasks:

    task history

search in list of complete tasks:

    task history searchterm

installation

Task is a single shell script. Just put it on your path and you are done.

It is also a standard practice to utilise two aliases for task. One representing a task file in your current directory, and one representing a global task file in your home directory. The current convention for these aliases is `x` for the local instance of task, and `xx` for the global instance of task. 

    alias x=task    
    alias xx="TASK_FILE=$HOME/.task x"
